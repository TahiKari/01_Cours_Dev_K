Créer et assembler des composants Vue

Les composants Vue

Les composants sont les briques essentielles de Vue.js. On peut avoir un composant pour une barre d’expérience, pour un footer, pour un bouton qu’on utilise tout le temps, etc.

Leur intérêt est qu’ils sont réutilisables.

Et si on a progressé dans cette habitude on pourra cliquer sur « + » pour gagner des points, sinon sur « - » pour en perdre.

La syntaxe Mustache

La syntaxe Mustache est une syntaxe de modèle qui permet d’afficher dynamiquement des données dans une page web en utilisant des modèles préformatés.

En Vue.js, la syntaxe Mustache est utilisée pour lier des données entre le modèle et les propriétés de l’instance de Vue. Pour utiliser la syntaxe Mustache dans un fichier HTML, on encadre la propriété de l’instance de Vue que l’on veut afficher avec des doubles accolades {{}}.

Les props

Les props en Vue.js sont utilisées pour transmettre des données d’un composant parent à un composant enfant. Cela permet de rendre les composants plus modulaires et réutilisables.

Les props sont définies en tant qu’attributs dans la balise du composant enfant, et sont passées au composant depuis le parent en utilisant la syntaxe v-bind. Le composant enfant peut alors utiliser ces props pour afficher des données ou effectuer des calculs.

Les computed properties

Les computed properties, ou propriétés calculées, sont un autre outil important de Vue.js pour la manipulation de données. Ils permettent de créer des propriétés qui dépendent de la valeur d’autres propriétés, mais qui sont calculées dynamiquement à partir de ces dernières. Les computed properties sont des fonctions qui retournent une valeur calculée en fonction de l’état actuel de l’instance Vue. Ces fonctions sont appelées automatiquement dès que les données auxquelles elles font référence sont mises à jour. Elles sont très utiles pour effectuer des opérations de calcul ou de filtrage sur les données de l’application, sans avoir à stocker les résultats dans des propriétés de données.

Par exemple, vous pouvez utiliser une computed property pour calculer le prix total dans un panier d’achats en fonction du nombre de chaque article et de son prix. Les computed properties sont similaires aux propriétés de données, mais avec une différence importante : elles ne sont pas modifiables directement. Au lieu de cela, elles sont mises à jour automatiquement lorsque les données auxquelles elles font référence changent. Cela signifie que les computed properties sont toujours à jour et reflètent l’état actuel de l’application. Il est important de noter que les computed properties sont mises en cache et ne sont re-calculées que lorsque les données auxquelles elles font référence sont mises à jour. Cela signifie qu’elles sont très performantes et qu’elles ne ralentissent pas l’application. En résumé, elles sont un moyen pratique et performant de calculer des valeurs dynamiques dans une application Vue.js. Elles permettent de séparer la logique de calcul de la présentation des données, rendant ainsi l’application plus facile à comprendre et à maintenir.

Les méthodes

Les méthodes en Vue.js sont un moyen de définir des fonctions que l’on peut appeler à partir d’un composant. Les méthodes sont définies dans l’objet « methods » d’une instance Vue, et peuvent être appelées depuis le code HTML du composant à l’aide de la syntaxe v-on. Les méthodes sont très utiles pour définir des comportements dynamiques dans un composant. Par exemple, si vous souhaitez effectuer une action lorsque l’utilisateur clique sur un bouton, vous pouvez définir une méthode qui gère l’événement de clic.

Les watchers

Les watchers sont un outil important de Vue.js pour suivre les changements de données dans l’application. Ils permettent d’exécuter du code en réponse aux changements de données, sans avoir besoin de re-rendre tout le composant. Ils sont définis dans l’objet « watch » d’un composant Vue. Lorsqu’une donnée référencée par le watcher change, le code défini dans le watcher est exécuté.

Voici un exemple de watcher dans un composant Vue :

See the Pen Untitled by OpenSpirit (@OpenSpirit) on CodePen.

Dans cet exemple, le watcher surveille la propriété « inputValue ». Lorsque cette propriété change, le code défini dans le watcher est exécuté. Ici, il met à jour la propriété « message » avec la nouvelle valeur de « inputValue ». Les watchers peuvent également être utilisés pour surveiller des propriétés calculées, des propriétés de Vuex ou d’autres données dans l’application. Il est important de noter que les watchers peuvent ralentir les performances de l’application s’ils sont mal utilisés. Il est recommandé d’utiliser les computed properties plutôt que les watchers lorsqu’il est possible de le faire, car les computed properties sont plus performantes.
