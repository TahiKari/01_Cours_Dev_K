Les techniques avancées pour passer les props avec React

Qu’est que le spread operator ?

Définition

Le « spread operator » (ou opérateur de diffusion en français) est un opérateur de syntaxe, utilisé en JavaScript et d'autres langages de programmation modernes, qui permet de « disséminer » (ou « étaler ») les éléments d'un objet itérable (tel qu'un tableau ou une chaîne de caractères) dans un autre objet.

En utilisant l'opérateur de diffusion, nous pouvons facilement concaténer deux tableaux, copier les propriétés d'un objet dans un autre objet, ou encore créer une copie d'un objet itérable avec une ou plusieurs de ses propriétés modifiées.

L'opérateur de diffusion est représenté par trois points de suspension ( " ... ") placés devant l'objet itérable à étaler.

Qu’est-ce que la déstructuration ?

Définition

En React, la déstructuration (ou « destructuring » en anglais) est une technique qui permet d'extraire des valeurs d'un objet ou d'un tableau et de les affecter à des variables distinctes, en utilisant une syntaxe simplifiée.

La déstructuration est couramment utilisée en React pour extraire des propriétés d'objets ou des éléments de tableaux, afin de les utiliser dans la création de composants.

Lorsqu'on utilise la déstructuration pour les props dans un composant React, cela permet d'extraire les données spécifiques qui nous intéressent des props et de les stocker dans des variables distinctes. Cela rend la lecture du code plus facile et permet d'éviter d'avoir à utiliser à plusieurs reprises l'objet props.

Gestion des props dans un composant de type classe

Dans un composant de type classe React, les props sont passées en tant que paramètres lors de la création de l'instance du composant et peuvent être accessibles via « this.props ».

Par exemple, si vous avez un composant parent qui rend un composant enfant " Message.js ", vous pouvez passer une props " text " au composant Message pour afficher un message personnalisé.

Dans ce cas, la props " text " est accessible dans le composant " Message.js " via « this.props.text ».

Dans certaines situations, vous aurez besoin d'accéder aux props dans le constructeur du composant. Pour cela, vous pouvez passer les props au constructeur en tant que paramètre et appeler " super(props) " pour permettre au composant de bénéficier des fonctionnalités du constructeur de sa classe parente.
