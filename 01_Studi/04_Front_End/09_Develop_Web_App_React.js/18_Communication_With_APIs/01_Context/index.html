Contexte

Lorsque l’on code une application React, il est très courant de devoir communiquer avec des serveurs tiers, par exemple afin de récupérer des données mises à disposition par quelqu’un d’autre, en vue de les intégrer à notre application. À cette fin, on utilise des API, ce qui désigne de manière générale les interfaces qui permettent aux applications ou aux blocs de code de communiquer entre eux.

Historiquement, la manière la plus commune de communiquer avec des serveurs tiers a longtemps été l’utilisation de l’API XMLHttpRequest, introduite en 1998 dans les navigateurs de l’époque. Progressivement, diverses librairies se sont construites sur cette API XMLHttpRequest, dont la très populaire librairie Axios. En parallèle, depuis 2015, l’API Fetch a été introduite dans les nouveaux navigateurs comme alternative à XMLHttpRequest. La plupart des développeurs web modernes utilisent soit Axios soit l’API Fetch, Axios présentant l’avantage d’être généralement plus simple pour les utilisations avancées, et l’inconvénient d’alourdir l’application par rapport à la solution native qu’est Fetch. Les applications React sont neutres de ce point de vue : elles sont parfaitement compatibles avec l’ensemble des solutions.
