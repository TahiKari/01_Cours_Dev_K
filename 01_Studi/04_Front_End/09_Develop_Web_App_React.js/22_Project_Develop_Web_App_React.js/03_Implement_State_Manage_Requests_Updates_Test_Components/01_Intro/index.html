Introduction

Objectifs

  • Mettre en place la gestion de l’état global de l’application en utilisant Redux. Cela permettra de stocker la ville saisie par l’utilisateur et la langue choisie.
  • Créer des slices (tranches) pour gérer l’état de la ville et de la langue. Chaque slice aura un reducer pour mettre à jour l’état et des actions pour modifier les valeurs.
  • Fournir le store de Redux à l’application en utilisant le Provider de React-redux dans le fichier « index.js ».
  • Effectuer des requêtes vers les API OpenWeatherMap et TimeZoneDb pour récupérer les données météorologiques et temporelles en temps réel de la ville. Utiliser Axios pour effectuer les requêtes et mettre à jour les données dans le composant Meteo.
  • Gérer le changement de langue dans l’application en permettant à l’utilisateur de sélectionner la langue souhaitée. Mettre à jour l’état de la langue et afficher les éléments de l’interface dans la langue sélectionnée.
  • Gérer l’entrée utilisateur en permettant à l’utilisateur de saisir le nom de la ville dans un input. Mettre à jour l’état de la ville et effectuer une nouvelle requête pour afficher les informations météorologiques de la nouvelle ville saisie.
  • Réaliser des tests pour chaque composant de l’application. Tester le rendu des composants, les fonctionnalités de changement de langue, de saisie de la ville et les appels aux API. Utiliser des bibliothèques de tests comme React-testing-library et Jest pour effectuer les tests.
