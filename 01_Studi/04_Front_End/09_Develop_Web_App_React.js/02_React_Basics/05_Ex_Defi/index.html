Exercice défi

Vous travaillez pour une jeune entreprise qui souhaite utiliser React pour développer votre application web. Pour commencer, vous devez décider si vous allez utiliser le CSR ou le SSR pour le rendu de l’application. L’entreprise veut une application fluide et interactive pour les utilisateurs.

Question

Décrivez brièvement les avantages et les inconvénients du CSR et du SSR. Expliquez ensuite quelle approche vous avez choisie et pourquoi.

Solution

L'avantage principal du CSR est le chargement initial de l'application rapide, car seule une page HTML minimale est envoyée au navigateur. Ensuite, le JavaScript est utilisé pour rendre l'application interactive et dynamique. Cependant, le principal inconvénient du CSR est que cela peut entraîner un temps de chargement plus long pour les pages contenant une grande quantité de contenu.

Le SSR, quant à lui, est une technique de rendu dans laquelle le code JavaScript est exécuté sur le serveur pour créer la page web. Cette approche peut réduire le temps de chargement initial de l'application, car le serveur peut générer une page HTML complète avec le contenu de l'application. Cependant, le principal inconvénient du SSR est que cela peut entraîner une complexité accrue de la mise en place et de la maintenance de l'application.

Pour ce travail, j'ai choisi d'utiliser le CSR pour le rendu de mon application. Cela est dû au fait que je veux une application qui soit rapide et interactive pour les utilisateurs, avec des temps de chargement aussi courts que possible.

L'avantage de l'utilisation de React via CDN est que vous n'avez pas besoin de télécharger ou d'installer la bibliothèque sur votre serveur. Le CDN s'occupera de la distribution de la bibliothèque à partir de serveurs répartis dans le monde entier, ce qui peut accélérer le chargement de votre application pour les utilisateurs qui se trouvent loin de votre serveur.

De plus, le CDN peut fournir des avantages de sécurité.
