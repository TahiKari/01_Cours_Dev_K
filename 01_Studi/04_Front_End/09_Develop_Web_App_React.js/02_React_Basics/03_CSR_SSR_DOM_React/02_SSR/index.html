Le SSR

Qu’est-ce que le SSR ?

Définition

Le SSR (Server-Side Rendering) est une technique de rendu de pages web qui permet au serveur de générer le code HTML complet d'une page web et de le renvoyer au navigateur de l'utilisateur. Cette technique est utilisée pour améliorer les performances et l'expérience utilisateur des sites web, notamment en réduisant les temps de chargement des pages.

Le SSR diffère du rendu côté client (CSR), dans lequel le navigateur du client doit télécharger le code HTML de base, puis exécuter du code JavaScript pour générer le contenu final. Avec le SSR, le serveur génère le HTML final et le renvoie directement au navigateur, ce qui peut réduire le temps de chargement initial de la page et améliorer la convivialité pour les utilisateurs ayant une connexion Internet lente ou limitée.

En résumé, le SSR est une technique de rendu de pages web qui permet de générer le contenu HTML côté serveur, avant de le renvoyer au navigateur de l'utilisateur, améliorant ainsi les performances et l'expérience utilisateur des sites web.

Le SSR avec React

ReactDOMServer est une partie de la bibliothèque React qui permet de rendre des composants React côté serveur. Il fournit des méthodes qui permettent de générer du HTML ou du SVG à partir de composants React, afin qu'ils puissent être pré-rendus avant d'être envoyés au navigateur.

Dans le contexte de React, le SSR est généralement implémenté à l'aide de bibliothèques tierces telles que Next.js ou Gatsby.js. Ces bibliothèques prennent en charge le rendu côté serveur de React, en générant le HTML côté serveur avant de l'envoyer au navigateur.

Le processus de SSR avec React se déroule en plusieurs étapes :

  • Le navigateur effectue une demande pour une page web, qui est reçue par le serveur.
  • Le serveur détermine quelle page web doit être rendue, en fonction de l'URL de la demande.
  • La bibliothèque de SSR (comme Next.js ou Gatsby.js) récupère les données nécessaires à partir de l'API ou de la base de données, puis rend l'application React côté serveur.
  • Le HTML généré est renvoyé au navigateur, qui peut alors l'afficher.

Une fois que le navigateur a reçu le HTML, il charge le JavaScript de l'application React et hydrate la page, ce qui permet à l'application de fonctionner en tant qu'application web normale.

En utilisant le SSR avec React, les avantages incluent une amélioration de la performance, une meilleure accessibilité pour les moteurs de recherche, et une expérience utilisateur plus rapide et plus fiable. Cependant, il peut y avoir des coûts de développement supplémentaires pour la mise en place du SSR, et il est important de comprendre les limites et les avantages de cette technique avant de l'utiliser pour votre application React.
